# install-proxmox

Installs Proxmox VE on bare-metal hosts tagged `role_proxmox_server` in the Ansible
inventory. Hosts must be running **Debian 12 (bookworm)** or **Debian 13 (trixie)** —
set `deploy_distro: trixie` (or `bookworm`) and `cloud_init_user: debian` in the
maas-pkg YAML for any machine that should run Proxmox.

- Debian 12 (bookworm) → Proxmox VE 8
- Debian 13 (trixie) → Proxmox VE 9

## Prerequisites

- MaaS has deployed Debian 13 (trixie) to the target host (wave `maas.machine.config.power`).
- Host is reachable via SSH (inventory generated by `generate_ansible_inventory`).
- Internet access from the host (Proxmox packages downloaded via apt).

## Commands

| Flag | What it does |
|---|---|
| `--build` | Generate inventory, install Proxmox VE, then verify once. |
| `--test` | Verify Proxmox is running; loops until all checks pass or `_PROXMOX_VE_WAIT_TIMEOUT` is reached. |
| `--deps` | Install Python and Ansible requirements only. |
| `--clean` | Remove generated tmp files. |
| `--clean-all` | Remove tmp files and the Python virtual environment. |
| `--status` | Show the current inventory file. |

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `_PROXMOX_VE_WAIT_TIMEOUT` | `600` | Seconds `--test` will poll. `0` = skip; `-1` = infinite. |

## Playbooks

### `playbook.install-proxmox.yaml`

Installs Proxmox VE on `proxmox_server` inventory group hosts. Steps:

1. Assert OS is Debian 12 (bookworm) or Debian 13 (trixie).
2. Resize root filesystem to fill the partition (cloud images are deployed undersized).
3. Mount EFI partition if missing from `/etc/fstab` (required before any GRUB operations).
4. Set hostname and `/etc/hosts` so the hostname resolves to the primary LAN IP (required by PVE).
5. Download Proxmox GPG key from `enterprise.proxmox.com`.
6. Add the `pve-no-subscription` apt repository; disable the enterprise subscription repository.
7. Remove MaaS curtin apt proxy (`90curtin-aptproxy`) — blocks Proxmox repo access — and
   deduplicate apt sources (trixie genericcloud writes both `debian.sources` and `sources.list`).
8. `apt full-upgrade` all packages.
9. Install `proxmox-default-kernel`; reboot if kernel changed or running kernel is not PVE.
10. Install `proxmox-ve postfix open-iscsi chrony`.
11. Remove Debian default kernel (`linux-image-amd64` and any versioned `linux-image-*` packages).
12. Run `update-grub`; remove `os-prober`.
13. Enable and start `pveproxy`, `pvedaemon`, `pvestatd`.
14. Mask `systemd-networkd` (do **not** stop — stopping drops the SSH connection by releasing
    the DHCP lease). Write an `auto <NIC> / iface <NIC> inet dhcp` fallback to
    `/etc/network/interfaces` so ifupdown2 brings the NIC up after the configure-proxmox
    reboot. Skipped if `vmbr0` already has a static IP (configure-proxmox has already run).
15. Copy the cloud user's SSH public key to `/root/.ssh/authorized_keys` without the
    forced-command prefix that cloud-init injects (Proxmox and configure-proxmox require
    direct root SSH access).

### `playbook.install-proxmox-test.yaml`

Verifies Proxmox VE is running. Called by `--test` in a polling loop.

Checks:
- `pveversion` exits 0 and contains `pve-manager`.
- `pveproxy`, `pvedaemon`, `pvestatd` services are active.
- Port 8006 (Proxmox web UI) is reachable from the controller.

## YAML configuration

Add the following to the maas-pkg YAML for any machine that should run Proxmox:

```yaml
"maas-pkg/_stack/maas/examples/example-lab/machines/<machine-name>":
  additional_tags: ["role_proxmox_server"]
  cloud_init_user: debian        # MaaS Debian deployment default user
  deploy_distro: trixie          # Debian 13 (trixie) → PVE 9; use bookworm for PVE 8
```

## Integration with `./run --build`

This script is called by the `hypervisor.proxmox.install` wave, after the
`maas.machine.config.power` wave deploys Debian 13 to the target machines. The wave:

1. Runs `proxmox-install-precheck` to verify all `proxmox_server` hosts are reachable via SSH.
2. Applies the Terragrunt units (which invoke this script as a Terraform hook).
3. Runs `proxmox-install-test` to verify `pveproxy`, `pvedaemon`, `pvestatd` are active
   and port 8006 is reachable on all nodes. Up to 6 retries with 120 s between attempts.

If no hosts are in the `proxmox_server` inventory group, the Ansible plays run with
zero targets and succeed immediately (no-op).
